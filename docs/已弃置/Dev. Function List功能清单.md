# Dev. Function List功能清单

> 本文档由Skyc8266创建，仅供开发使用！

## A. 用户功能

### A1. 记忆系统

- 针对不同角色分别管理记忆，使用SQLITE数据库存储

#### A1.1 用户管理与自动备份

- **自动备份受保护模块管理**
- 用户可以手动管理每个角色的记忆条目

#### A1.2 系统自动记忆管理与强制对话调用

- 每个角色都有独立的记忆配置（例如**记忆力**与其他自定义配置），依据这些配置，系统会自动决定其长期记忆量（可变）、执行不同的记忆条目调用需要多长时间与短期记忆的遗忘速度等
- Ai在回复中涉及到任何过往信息的，必须从记忆库中调取，多数长期记忆与遗忘程度极低的短期记忆将会在每次开启新对话时自动载入到上下文中，最大载入量受角色记忆配置影响

### A2. OpenAi兼容的Ai模块

- 这个模块提供一系列OpenAi兼容的Api，用于支持隙间的ai对话，支持多模态与工具调用等，支持音频、视频与图像

### A3. 角色与状态系统

- 本系统用于角色与其状态的统一管理

#### A3.1 角色信息系统

- 管理角色文档与过往记忆**（属于记忆系统，但不可被修改）**，并管理其Live2D或3D模型（包含贴图）与基本动作库，同时，管理其声音数据与可能的笔迹（用于视频生成）数据、语言风格等各类数据
- 本系统会自动缓存部分内容，以实现在对话或多模态内容生成时的快速响应，包括笔迹数据、多角度形象图、模型贴图等各类常用数据

#### A3.2 角色状态系统

- 统一管理角色的饱食、饮水、健康、心情等状态信息，并随时间与模拟世界内状态修改其中的一些值，自动修改受角色配置与角色有关文档影响

### A4. 模拟世界系统

- 管理模拟世界

#### A4.1 突发/常见时间自动管理

- 系统内Ai主动创建事件，并绑定至对应世界，同时在需要的情况下生成场景
- 本功能受用户管理，且被角色对话与世界配置、世界文档影响

#### A4.2 世界总管理

- 同时维护多个世界，世界通过世界观文档等设计，系统内会自动创建大量的**“配角”**，并为他们分配少量算力资源用于模拟世界
- 自动设置天气、时间、光照、环境等内容，并维护在SQLITE数据库中
- 允许用户重置、小幅度修改世界，世界内的一切都会被记录，部分与角色有关的内容将会被载入记忆系统，世界内的一切会对配角造成影响，同时也会影响角色与用户。

#### A4.3 场景与互动

- 用户可以选择不同的交通方式前往模拟世界内各个地点，进行不同的互动，并对角色与世界（如果必要）造成影响

#### A4.4 经济系统（允许用户小幅度修改）

- 可以通过不同方式（例如**卖东西**、**抢银行**）获取模拟世界内的某种货币（例如原神世界观内的摩拉、崩铁世界观内的信用点），并购买物品或进行其他交易操作（**注**：同一模拟世界内的其他角色（包含配角）可以偷走你的货币！你也有可能被诈骗！）
- 世界经济总系统（模拟）

### A5. 安全模块

#### A5.1 模型输出审查

- 对模型的输出进行实时监控，防止其OOC（特殊情况下可以放松审查，例如在模拟世界下正在镜流极度危险事件时，平时冷静的角色也可以表现出强烈的害怕、不安等情绪），同时，在模型开始生成前，先对用户的输入进行检查，防止提示词注入等恶意行为。（**注**：可以去网上查下崩铁的帕姆ai，那个防的够严）

#### A5.2 电脑控制防护

- 自动实时判断危险/越界操作并及时断开MCP连接
- 当用户按下指定的安全终止快捷键时，立即冻结MCP服务进程，并将模型从内存强制弹出。完成后备份当前上下文到专用备份文件夹，若用户确认，保护模块开始对上下文进行删改，以达到去除危害性内容/指令的目的，随后重启MCP服务并加载Ai到内存

#### A5.3 自动世界、记忆上下文备份

- 模块会自动保存快照，用户可限制备份文件的最大占用空间，达到限制时提示，若用户同意，则压缩快照。

#### A5.4 系统过载防护（禁止关闭）

- 防护模块会自动监控系统占用、传感器数据等信息，并在模块认为系统接近过载时，强行终止Ai回答并释放内存，随后将当前上下文等数据备份，完成后提示用户，若用户**双重**确认，则在20秒后（为设备提供短暂的恢复时间）重启ai服务并重载上下文。
- 过载防护**不可关闭**（为了保护设备安全，防止过热、过载等，防止硬件损坏、系统崩溃），但用户可以调整其档位（两档，默认严格，可选适中），这个防过载可以参考LMStudio的过载保护

### A6. 实时通话

- 用户可以与角色实时**语音**通话，角色可以通过其Live2D（或3D）模型做出动作回应（也可以设置特效/简短动画），角色也可以在通话中歌唱～。

### A7. 主动发起聊天或通话（仅支持macOS与iOS/iPadOS）

- 本功能需要保持“隙间”在后台运行
- 角色可自主发起语音通话或是互动/聊天等，随后用户设备上会出现通知或通话接听提醒

### A8. 将角色（可多个）作为桌宠或动态壁纸（仅支持macOS）

- 用户可将一个或多个角色设置为桌宠，桌宠可持续在用户屏幕上活动（部分角色可以飞行，在配置文件中设置），在**用户允许**的情况下桌宠可以操作电脑、移动窗口、鼠标点击、键盘输入等（可以在**合理范围**内捣乱）
- 用户可将单个角色设置为动态背景，背景为模拟世界内的场景（有随时间变化等功能，有环境实时模拟（如果必要）），角色可在其中随意活动，并在**用户允许**的情况下，改变桌面布局、移动窗口，但**无法进行任何其他操作**。
