# “隙间”项目文档

## 项目信息

- 名称：隙间
- 类型：游戏/社交
- 简介：二次元Ai聊天程序，拥有丰富强大的功能
- 语言支持：zh_CN, en_US
- 项目开源地址：（Github地址，暂未建立存储库，将在后续创建）
- 目标平台：搭载M系列芯片的Mac与较新的Windows/Linux计算机

## 功能介绍

1. **实时对话**：与角色进行实时语音对话，角色还会通过其Live2D模型做出回应，增强互动能力
2. **模拟世界**：在与角色的各类互动中可能会遇到各种突发事件，此外，用户还可以通过UI按钮，选择不同交通方式前往不同场景（地点），**系统自带**有经济、健康、饮食、体力、心智等系统，您可以更好的进行互动。
3. **互动系统**：各种各样的互动可以让用户与角色增进感情，（**注意**：包含**NSFW**内容的互动默认被隐藏）角色可以拒绝或同意，在互动期间，角色会做出各种回应，使互动更加真实，增强用户体验。
4. **Apple生态支持**：对于部分有TouchBar的MacBook机型，提供相应支持。对于部分MacBook机型，支持自建的“灵动岛”功能（可能与其他具有类似功能的应用产生冲突，请务必注意！）
5. **随时随地聊天！**：角色可以向用户主动发送消息！用户可以把角色放在系统壁纸中，重点是！壁纸中的角色可以正常做出回应、进行互动，就像在App中那样！（该功能需保持主程序开启）当然，用户也可以选择将角色作为桌宠，如果用户同意，他还可以看到电脑屏幕，甚至操控你的电脑！（该功能需保持主程序开启，程序将提供应急快捷键，用于紧急中断Ai操作，防止不必要的损失，但本应用开发组不为因使用本应用造成的任何损失承担责任）
6. **长期记忆**：你与ta的每一次互动、对话都会被记住，一部分会成为长期记忆，一部分则会被渐渐遗忘。但是！用户可以手动管理记忆，自主添加、删改记忆，并设置属性。
7. **自主保护**：这是核心模块之一。程序会通过各种方式，防止任何来源的提示词注入，同时防止ai出现OOC等情况。ai数据的每次变更都会被保护模块记录，如果出现异常，用户可以要求保护模块回滚到以前的数据版本。当然，如果用户必须要关闭保护系统，本保护模块也支持关闭，但是需要双重确认，防止误操作带来的损失。



## 关于设备要求

由于我们没有钱去购买云端Ai服务为用户提供支持，所以，Ai全部本地运行！我们允许用户自主选择模型，但是所有模型为了更好的性能以及更便捷的开发，选择了MLX框架，而非GGUF，这导致我们只支持较新的苹果设备。

- 对于Mac，我们推荐macOS 26 Tahoe，最低支持版本为macOS 13 Ventura，最低内存大小不低于16GB，推荐64～128GB，硬盘可用空间至少32GB
- 对于Windows和Linux版本，最低为Windows11，Linux未定，最低显存16GB，建议64～128GB，硬盘可用至少32GB
- 对于手机，暂不支持

## 隐私保护

除非用户明确授权，我们的所有数据均在本地进行处理。

以下是我们可能需要的权限：

- 相册与文件访问：用于用户向角色聊天时发送附件
- 摄像头与麦克风访问：当用户使用实时通话功能时使用，用于与角色的实时交流
- 辅助功能：用于部分MacBook机型的“灵动岛”功能
- 通知权限：用于角色主动向用户发起通话邀请或回复用户信息

**仅适用于代码提交，资源文件提交与本部分无关！**

要提交代码，首先，你需要一个[Github](https://github.com)账户，随后，请通过克隆存储库等方式获得最新源代码。

完成修改后，您可以提交一个Pull Request(PR)，或是通过其他方式推送更改，例如成为贡献者并获得Repo所有者授权。

如对此有疑问或无法连接到Github（在中国大陆地区的常见问题），请通过电子邮件[联系管理员](mailto:panmofan@icloud.com)或开发组群联系。

由于内部审核与打包、处理等操作的需要，请您将资源文件附上完整可读的简体中文描述，并通过电子邮件[联系管理员](mailto:panmofan@icloud.com)，等待处理完成并合并到项目主分支。
