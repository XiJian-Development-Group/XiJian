## 功能介绍

**注意**：如果要获取详细的功能清单，请查看"docs/Dev. Function List功能清单v2.md"

1. **实时对话**：与角色进行实时语音对话，角色还会通过其3D模型做出回应，增强互动能力
2. **模拟世界**：在与角色的各类互动中可能会遇到各种突发事件，此外，用户还可以通过UI按钮，选择不同交通方式前往不同场景（地点），**系统自带**有经济、健康、饮食、体力、心智等角色状态系统，您可以更好的进行互动。
3. **互动系统**：各种各样的互动可以让您与角色增进感情，（**注意**：包含**NSFW**内容的互动默认被隐藏）角色可以拒绝或同意，在互动期间，角色会做出各种回应，使互动更加真实，增强用户体验。
4. **主动聊天**：角色可以向用户主动发送消息！用户可以把角色放在系统壁纸中，重点是！壁纸中的角色可以正常做出回应、进行互动，就像在App中那样！（该功能需保持主程序开启，且设备性能支持本程序在后台进行活动）当然，您也可以选择将角色作为桌宠，如果您同意，他还可以看到电脑屏幕，甚至操控你的电脑！（该功能需保持主程序开启，程序将提供应急快捷键，用于紧急中断Ai操作，防止不必要的损失，但本应用开发组不为因使用本应用造成的任何损失承担责任）
5. **长期记忆**：你与ta的每一次互动、对话都会被记住，一部分会成为长期记忆，一部分则会被渐渐遗忘。但是！用户可以手动管理记忆，自主添加、删改记忆，并设置属性。
6. **自主保护**：这是核心模块之一。程序会通过多种方式，防止绝大多数提示词注入，同时防止ai出现OOC等情况。ai数据的每次变更都会被保护模块记录，如果出现异常，用户可以要求保护模块回滚到以前的数据版本。当然，如果您必须要关闭保护系统，但是需要双重确认，以防止误操作带来的损失。此外，当系统过载时，保护模块也会强制终止Ai模块与相关进程。

## 设备支持

**操作系统与版本**：macOS >= 26.0, Windows >= 24H2, iOS >= 26.0, Android >= 12.0

**最低配置**：

对于Mac设备，M系列芯片是必须的，16GB内存为最低，其中可用内存不低于6GB，硬盘可用空间不低于24GB

对于Windows设备，要求略有提高，最低20GB可用显存，硬盘建议为固态硬盘，且可用空间不低于26GB

对于iOS设备，您需要连接到一个可以正常被您访问的，运行了隙间最新版本的计算机，才能正常使用全部功能。

---

**解释**：

本程序在本地运行Ai，故对设备要求较高。对Windows的配置要求比Mac略高是因为，隙间在实现Core Api与Ui App时，为了
跨平台等原因，在Windows与Mac的实现上使用了不同的逻辑与框架。

## 项目特点介绍

### 本地Ai

您与角色互动时，数据将始终保持在本地，因为我们默认**在本地运行Ai**。所以，这将对设备配置有较高要求。

**解释**：我们没有钱去购买云端Ai服务为用户提供支持，也不希望用户的数据被上传到云端，所以决定在本地运行全部Ai。
但是，有些用户的设备配置明显是不足以本地运行Ai的，因此，我们在未来版本也会允许用户连接到OpenAi兼容的Api服务，
来正常体验Ai功能

### 真实的互动

本项目在设计之初就是为了更真实的互动。为了这一点，我们添加了记忆系统、模拟世界以及其他多个模块，
希望这能为大家带来更好的体验，希望大家能够向我们发送使用反馈，以便我们优化项目功能。
