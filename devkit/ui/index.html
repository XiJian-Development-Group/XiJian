              开发者工具使用概述

              隙间开发者工具是一套本地创作套件，帮助你创建、编辑角色人设、记忆条目、世界观文档、声音样本和 3D 模型，并将它们打包提交给审核团队。

              工作流程

              1. 登录 — 在「创作提交」标签页输入你的开发者 ID。
              2. 创作内容 — 在对应标签页中创建角色、记忆、世界观等内容。
              3. 选择提交包 — 切换到「创作提交」，从可提交列表中勾选要提交的内容。
              4. 提交 — 填写 AI 协助占比和补充说明，点击提交。系统会自动打包并通过邮件发送。

              注意事项

              • 每个开发者 ID 每 1 小时只能提交一次。
              • 附件总大小不超过 1200 MB。
              • 所有数据存储在本地 ~/Library/Application Support/XiJian/DevKit/ 目录下。

              角色人设

              在此管理角色的基本信息、人设文档和相关配置。

              字段说明

              名称
              角色的唯一标识名，例如 yuki。创建后不可更改。
              显示名称
              在对话中显示的名称，例如「雪」。
              语言风格
              角色说话的风格，例如 modern_jp（现代日语）、classical（古典）、tsundere（傲娇）。可自定义。
              默认情感
              角色默认的情感状态，如 neutral、happy、sad。
              标签
              逗号分隔的关键词，用于分类和检索。
              人设文档
              用 Markdown 编写角色的完整人设，包括背景故事、性格、喜好等。支持从本地的 .md 文件导入。
              关联配置
              将角色与已有的世界观、记忆包、声音包或 3D 模型关联。提交时会一同打包。
              记忆配置
              控制角色的记忆系统行为：长期/短期记忆上限、重要性阈值、上下文 Token 限制等。

              记忆条目

              为角色编写长期或短期记忆片段，构成角色的知识库。

              字段说明

              角色 ID
              记忆所属的角色 ID。
              类型
              短期：对话上下文中临时存储的信息；长期：被固化到角色知识库中的重要信息。
              重要性 (0.0–1.0)
              该记忆的重要程度。重要性 ≥ 长期阈值的记忆才会被写入长期存储。
              标签
              逗号分隔的关键词，用于分类。
              记忆内容
              记忆的具体描述。应当清晰、具体，便于模型理解。

              打包提交

              一个角色的全部记忆条目会作为一个「记忆包」整体导出。在提交标签页的列表中选择对应角色 ID 的记忆包即可。

              世界观

              创建和管理世界设定文档。

              字段说明

              名称
              世界观的名称，例如「现代东京」或「奇幻大陆」。
              配置文件（JSON）
              可选的结构化配置，例如标签、规则等。
              世界观文档（Markdown）
              世界观的详细描述，包括历史、地理、社会结构、特殊规则等。

              声音克隆

              管理角色的声音参考样本。

              字段说明

              角色 ID
              声音所属的角色 ID。
              名称
              声音样本的名称，例如 default、whisper。
              声音引擎
              选择用于合成声音的引擎。支持：melo-tts, cosyvoice, gpt-sovits, fish-speech, chat-tts, bert-vits2, vall-e, openai-tts。
              音频文件路径
              参考音频文件（.wav / .mp3 / .m4a / .ogg / .flac）。

              创作提交

              将创作的内容打包并通过邮件提交给审核团队。

              流程

              1. 登录 — 输入你的开发者 ID，用于追踪提交记录。
              2. 选择内容类型 — 选择产出物类型并填写标识，或使用下方的包列表直接选择。
              3. 选择内容包 — 从「选择要提交的内容包」列表中勾选需要提交的项目。列表中会自动聚合所有已创建的角色、记忆包和世界观。
              4. 填写信息 — 设置 AI 协助占比和补充说明。
              5. 提交 — 点击提交按钮，系统会打包所有选中内容并通过邮件发送。

              包类型说明

              角色包 (character)
              包含角色的 JSON 配置和人设文档（persona.md）。
              记忆包 (memory_pack)
              包含指定角色的所有记忆条目，导出为 JSON 文件。
              世界观包 (world)
              包含世界观的 JSON 配置和文档。

              限制

              • 同一开发者 ID 每 3600 秒（1 小时）可提交一次。
              • 附件上限 1200 MB（超出会拒绝）。
              • 提交后会在左侧「历史提交」面板显示记录。
