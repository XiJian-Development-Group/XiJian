                                帮助文档

                                开发者工具使用概述

                                隙间开发者工具是一套本地创作套件，帮助你创建、编辑角色人设、记忆条目、世界观文档、声音样本和 3D 模型，并将它们打包提交给审核团队。所有数据在本地管理，创作完成后再通过邮件提交。

                                工作流程

                                1. 登录 — 在「创作提交」标签页输入你的开发者 ID。该 ID 用于追踪提交记录和冷却时间。
                                2. 创作内容 — 在对应标签页中创建角色人设、记忆条目、世界观等。每个标签页都配有独立的编辑器和列表。
                                3. 配置关联 — 在角色编辑器中，可以将角色与世界观、记忆包、声音包、3D 模型关联。关联后的内容会在提交时一同打包。
                                4. 选择提交包 — 切换到「创作提交」，从「选择要提交的内容包」列表中勾选需要提交的项目。系统会自动聚合所有已创建的角色、记忆包和世界观。
                                5. 填写提交信息 — 设置 AI 协助占比和补充说明。
                                6. 提交 — 点击提交按钮，系统会自动打包所有选中内容，生成 7Z 格式的归档文件，并通过 SMTP 邮件发送给审核团队。

                                注意事项

                                • 每个开发者 ID 每 1 小时只能提交一次（冷却机制）。
                                • 附件总大小不超过 1200 MB。
                                • 所有数据存储在本地 ~/Library/Application Support/XiJian/DevKit/ 目录下。
                                • 提交后可在左侧「历史提交」面板查看记录。
                                • 声音引擎、关联配置等下拉选项在启动时自动加载。

                                角色人设

                                在此管理角色的基本信息、人设文档和相关配置。角色是人设的核心，所有记忆条目、声音样本和关联配置都围绕角色展开。

                                字段说明

                                名称（必填）
                                角色的唯一标识名，用于内部识别。建议使用英文或拼音，例如 yuki、zhong_li。创建后不可更改。
                                显示名称
                                在对话界面中展示给用户的名称，例如「雪」或「钟离」。支持中文和其他 Unicode 字符。
                                语言风格
                                定义角色说话时的语言风格和口吻。常用值包括：modern_jp（现代日语风格）、classical（古典/文言风格）、tsundere（傲娇风格）、gentle（温柔风格）。也可以自定义任意文本。
                                默认情感
                                角色在无特殊上下文时的默认情感状态。推荐值：neutral（中立）、happy（开心）、sad（悲伤）、angry（生气）、worried（担忧）。
                                标签（逗号分隔）
                                用于分类和检索的标签，例如 tsundere, student, anime。标签会在角色列表中显示。
                                人设文档（Markdown）
                                用 Markdown 格式编写角色的完整人设。建议包含以下内容：
                                • 背景故事 — 角色的出身、经历、重要事件。
                                • 性格描述 — 性格特点、优缺点、行为模式。
                                • 喜好与厌恶 — 角色喜欢和不喜欢的事物。
                                • 说话方式 — 语气、口头禅、语速等特点。
                                • 人际关系 — 与其他角色的关系。
                                支持从本地 .md、.markdown 或 .txt 文件导入。点击「从 .md 文件导入」按钮选择文件即可。

                                关联配置

                                关联配置允许你将角色与已有的其他内容关联起来，提交时会一同打包。

                                关联世界观
                                选择角色所处的世界观。下拉列表会自动填充已在「世界观」标签页中创建的世界观。
                                关联记忆包
                                选择角色的记忆包来源。列表显示已有记忆条目的角色 ID。
                                关联声音包
                                选择角色的声音包来源。列表显示已有声音样本的角色 ID。
                                关联 3D 模型
                                选择角色的 3D 模型。列表自动填充已在「3D 预览」标签页中注册的模型。

                                记忆配置

                                控制角色的记忆系统行为。这些参数决定了模型如何处理和利用角色的记忆。

                                长期记忆上限
                                长期记忆中最多存储的条目数。超出后旧条目会被淘汰。默认 200。
                                长期记忆重要性阈值
                                记忆重要性达到此值（0.0–1.0）才会被写入长期存储。较高的值意味着只有非常重要的记忆才会被长期保留。默认 0.6。
                                短期记忆上限
                                短期记忆中最多存储的条目数。短期记忆在对话结束后会逐渐衰减。默认 50。
                                短期记忆衰减率
                                短期记忆每次衰减的比例（0.0–1.0）。较高的值意味着短期记忆衰减更快。默认 0.05。
                                短期重要性阈值
                                记忆重要性达到此值才会被纳入短期记忆上下文。默认 0.3。
                                上下文 Token 上限
                                模型在处理对话时，最多用于记忆上下文的 Token 数。默认 8000。
                                回复保留 Token
                                为模型生成回复预留的 Token 数。默认 2000。
                                强制召回历史
                                是否在每次对话时强制召回历史记忆。选择「是」则会始终考虑历史记忆。

                                记忆条目

                                为角色编写长期或短期记忆片段，构成角色的知识库和经历。记忆条目按角色 ID 分组管理。

                                字段说明

                                角色 ID
                                记忆所属的角色 ID。需要在「角色人设」标签页中先创建角色。也可以在角色编辑器中直接指定。
                                类型
                                短期：对话上下文中临时存储的信息，会随着时间衰减。适合记录对话中的临时话题。长期：被固化到角色知识库中的重要信息，不会自动衰减。适合记录角色的核心设定和重要经历。
                                重要性 (0.0–1.0)
                                该记忆的重要程度。重要性 ≥ 角色配置中「长期记忆重要性阈值」的记忆才会被写入长期存储。较高的值意味着这条记忆对角色行为影响更大。
                                标签（逗号分隔）
                                用于分类的关键词，例如 preference, food, memory_001。标签便于检索和过滤。
                                记忆内容
                                记忆的具体描述。建议清晰、具体，便于模型理解。例如：「雪非常喜欢吃草莓蛋糕，每次路过甜品店都会买一块。」

                                如何编写好的记忆

                                • 具体 — 避免模糊描述，给出具体的细节和情境。
                                • 一致性 — 确保记忆之间逻辑一致，不互相矛盾。
                                • 重要性分级 — 核心设定用高重要性（≥ 0.7），日常偏好用中等重要性（0.3–0.6），临时话题用低重要性（< 0.3）。
                                • 短期 vs 长期 — 长期记忆用于固定的角色设定；短期记忆用于对话中的临时信息。

                                打包提交

                                一个角色的全部记忆条目会作为一个「记忆包」整体导出。在提交标签页的列表中会显示为「<角色名> 的记忆包」，勾选即可提交。导出的记忆包为 JSON 格式，包含该角色的所有记忆条目。

                                世界观

                                创建和管理世界设定文档。世界观为角色提供背景环境，影响角色的行为和对话内容。

                                字段说明

                                名称（必填）
                                世界观的名称，例如「现代东京」或「奇幻大陆·艾尔索斯」。应简洁且具有辨识度。
                                配置文件（JSON）
                                可选的结构化配置，用于存储机器可读的设定参数。例如： 
                                {
  "tags": ["modern", "school", "tokyo"],
  "rules": "禁止使用魔法",
  "time_period": "2024年"
}
                                世界观文档（Markdown）
                                使用 Markdown 详细描述世界观。建议包含以下内容：
                                • 概述 — 世界的基本设定和风格。
                                • 地理 — 世界的地理环境、重要地点。
                                • 历史 — 重要历史事件和时间线。
                                • 社会结构 — 政治体制、阶级、文化习俗。
                                • 特殊规则 — 魔法、科技、超自然力量等特殊设定。
                                • 角色说明 — 该世界观中活跃的角色及其关系。

                                注意事项

                                • 世界观文档越详细，模型对世界背景的理解越准确。
                                • 配置文件中的 tags 可用于后续的筛选和匹配。
                                • 一个世界观可以关联多个角色。

                                声音克隆

                                管理角色的声音参考样本。声音样本用于训练或配置角色语音合成。

                                字段说明

                                角色 ID
                                声音所属的角色 ID。需要在「角色人设」标签页中先创建角色。
                                名称
                                声音样本的名称，例如 default（默认声线）、whisper（低语）、excited（兴奋）。一个角色可以有多个声音样本。
                                声音引擎
                                选择用于合成声音的引擎。支持以下引擎：
                                • melo-tts — 轻量级 TTS，适合快速生成
                                • cosyvoice — 自然度高的语音合成
                                • gpt-sovits — GPT 驱动的 SOVITS 模型
                                • fish-speech — 高质量语音合成
                                • chat-tts — 对话式 TTS
                                • bert-vits2 — BERT + VITS2 混合模型
                                • vall-e — 神经编解码语音合成
                                • openai-tts — OpenAI TTS API
                                音频文件路径
                                参考音频文件，用于声音克隆的样本。支持的格式：.wav、.mp3、.m4a、.ogg、.flac。可以通过「选择文件」按钮从本地选取。

                                最佳实践

                                • 使用高质量的参考音频（采样率 ≥ 44.1kHz，无杂音）。
                                • 参考音频应包含角色的典型说话方式。
                                • 不同引擎对样本质量的要求不同，建议查阅各引擎的文档。

                                剧情设计

                                设计故事的剧情大纲，使用节点（Nodes）和连线（Edges）构建故事流程。剧情设计采用可视化图结构，每个节点代表一个故事片段，连线定义节点之间的流转关系。

                                节点类型

                                对话
                                角色之间的对话片段。
                                叙述
                                旁白或场景描述。
                                选择
                                玩家/用户的选择点，不同选择走向不同分支。
                                事件
                                触发特定事件或剧情转折。
                                分支
                                条件分支节点，根据条件走向不同路径。

                                连线和标签

                                连线定义节点之间的流转关系。每一条连线从「来源节点」指向「目标节点」。可以为连线添加标签（例如「选择 A」「条件满足」），便于理解分支逻辑。

                                导出提交

                                剧情会整体导出为 JSON 格式，包含剧情元数据、所有节点和连线。在「创作提交」标签页中选择勾选即可提交。

                                对话样本

                                为角色编写对话样本数据，用于训练或配置角色对话模型。每个角色应有足够数量和场景覆盖的对话样本。

                                字段说明

                                角色 ID
                                对话所属的角色 ID。
                                场景
                                对话发生的场景分类，例如 greeting（问候）、farewell（告别）、combat（战斗）、daily（日常）等。良好的场景覆盖能提高角色对话的泛化能力。
                                用户消息
                                用户的输入内容。这是对话的触发信息。
                                角色回复
                                角色在该场景下应有的回复内容和语气。应体现角色的性格和语言风格。
                                情感标签
                                该回复的情感状态，例如 neutral（中立）、happy（开心）、sad（悲伤）、angry（生气）等。

                                最低要求检查

                                点击「检查最低要求」按钮可以检查当前角色的对话样本是否满足最低提交标准（通常需要至少 10 条对话样本，覆盖 3 种以上场景）。不满足最低要求的角色无法打包提交。

                                编写建议

                                • 覆盖多种常见对话场景（问候、告别、请求、拒绝、提问等）。
                                • 角色回复应符合角色人设中定义的语言风格和性格。
                                • 每条对话样本都应包含合理的用户消息和角色回复。

                                动作编辑

                                管理角色的动作/动画数据。可以为角色定义各种动作（行走、挥手、闲置等），并关联动作文件（BVH、FBX、GLB）。

                                字段说明

                                角色 ID
                                动作所属的角色 ID。
                                动作名称
                                动作的标识名称，例如 idle（闲置）、walking（行走）、waving（挥手）。
                                动作描述
                                对动作的具体描述，说明动作的内容和触发时机。
                                动作参数（JSON）
                                可选的结构化参数，用于控制动作的行为。例如时长（duration）、是否循环（loop）、融合因子（blend）等。
                                文件路径
                                动作文件的本地路径。支持 BVH、FBX、GLB、GLTF 格式。可以通过「导入」按钮从本地选择文件。

                                导入文件

                                点击「导入」按钮可以选择本地的动作文件。导入后文件路径会自动填入，并注册到角色的动作列表中。

                                AI 辅助

                                追踪 AI 辅助创作事件、计算 AI 使用率，并获取智能建议。这个面板帮助开发者了解 AI 在创作中的参与程度。

                                AI 事件记录

                                每次使用 AI 辅助（如 AI 生成、AI 建议、AI 辅助编辑、AI 翻译、AI 分析）时，记录一条事件。事件类型包括：

                                • AI 生成 — AI 完整生成的内容（如 TTS 语音生成、AI 模型生成）。
                                • AI 建议 — AI 提供建议，由人工确认后使用。
                                • AI 辅助编辑 — AI 辅助修改或优化现有内容。
                                • AI 翻译 — AI 翻译内容。
                                • AI 分析 — AI 分析现有内容并提供反馈。

                                AI 使用率

                                AI 使用率 = AI 生成事件数 / 总事件数。提交时可以设定 AI 协助占比（0–1），并在「AI 辅助」面板中检查是否超过设定的阈值。

                                智能建议

                                输入上下文文本，点击「获取建议」可以请求 AI 提供创作建议。建议内容依赖于当前已记录的事件和上下文。

                                阈值检查

                                设定 AI 使用率阈值后可以检查当前使用率是否超标。在创作提交时，平台可能会对 AI 使用率过高的内容进行额外审查。

                                创作提交

                                将创作的内容打包并通过邮件提交给审核团队。提交前请确保所有内容已保存。

                                提交流程（详细）

                                1. 登录 — 在页面顶部的登录区域输入你的开发者 ID。该 ID 用于追踪提交记录和计算冷却时间。建议使用固定的 ID 以保持提交历史连续。
                                2. 准备内容 — 确保你在角色人设、记忆条目、世界观等标签页中创建的内容已经保存。可以在各标签页中编辑和完善内容。
                                3. 选择内容包 — 在「选择要提交的内容包」区域，点击「刷新可提交列表」获取最新的可提交内容列表。列表中会显示：
                                  • 角色包 — 每个已创建的角色显示为一项，勾选后提交角色的 JSON 配置和人设文档（persona.md）。
                                  • 记忆包 — 每个有记忆条目的角色显示为一项，勾选后提交该角色的所有记忆条目（导出为 JSON）。
                                  • 世界观包 — 每个已创建的世界观显示为一项，勾选后提交世界观的 JSON 配置和文档。
                                  可以同时勾选多个包，它们会被打包到同一个归档中提交。
                                4. 填写提交信息：
                                  • AI 协助占比 (0–1) — 表示该内容的 AI 辅助创作比例。0 表示完全人工创作，1 表示完全 AI 生成。建议如实填写。
                                  • 补充说明（可选） — 任何你想告知审核团队的额外信息。
                                5. 提交 — 点击「提交」按钮。提交过程包括：
                                  1. 验证内容和冷却时间
                                  2. 打包为 7Z 归档文件（含 manifest.json 和所有内容文件）
                                  3. 计算文件 SHA256 校验值
                                  4. 通过 SMTP 发送邮件到审核邮箱
                                  5. 记录提交历史
                                  整个过程通常在 10–60 秒内完成。

                                包类型说明

                                角色包 (character)
                                包含角色的 JSON 配置（character.json）和人设文档（persona.md）。JSON 配置中包括角色的名称、显示名称、语言风格、默认情感、标签、关联配置和记忆配置等所有字段。
                                记忆包 (memory_pack)
                                包含指定角色的所有记忆条目，导出为 JSON 文件。注意：记忆包始终以角色 ID 为单位整体导出，不能只提交单条记忆。
                                世界观包 (world)
                                包含世界观的 JSON 配置文件（world.json）和文档（world_doc.md）。

                                限制与规则

                                • 冷却时间 — 同一开发者 ID 每 3600 秒（1 小时）可提交一次。冷却时间在成功提交后开始计算。
                                • 附件大小 — 所有选中内容打包后的归档文件不得超过 1200 MB。超出会拒绝打包。
                                • 邮件发送 — 使用 SMTP 协议通过发件服务器发送。发送失败会返回错误信息。
                                • 提交历史 — 成功提交后会在左侧「历史提交」面板显示记录，包含提交 ID、时间、大小和 SMTP 状态。
